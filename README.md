# Replication materials for: Inglese, Mazzola, Goria, Ferrarotti & Cornillie, “Anticausativization and lability in Romance: a historical corpus study on Spanish and Italian” in Inglese, Mazzola & Goria *Diachronic and Typological Perspectives on Anticausativization* (Accepted December 2025). 

Full citation information will be released after publication.

## Overview

This repository contains the code, data, and figures necessary to document and partially reproduce the analyses reported in the paper:

> **Anticausativization and lability in Romance: a historical corpus study on Spanish and Italian**

[Full paper citation will be added after publication]

The repository is organized to ensure transparency and long-term accessibility of the materials used in the study. Due to the use of an external interactive tool, **full computational reproducibility is not possible**, but all analytical steps are documented in detail.

---

## Repository contents

* **R code**

  * A Quarto report (`.qmd`) containing the main analyses, tables, and figures presented in the paper
  * Additional R scripts (`.R`) with functions created to automatised plots and reports
  
* **Data**

  * Input data files in `.xlsx` format

* **Figures**

  * Plots automatically generated by the Quarto document
  * Additional plots saved manually, produced by the standalone R script

---

## How to use this repository

### Requirements

* R (version ≥ 4.x recommended)
* RStudio (recommended)
* Quarto

All required R packages are loaded within the `.qmd` file. Missing packages can be installed manually if needed.

---

### Running the Quarto report

Make sure the `.qmd` document is in a folder that contains all the materials included in the repository.
Open the `.qmd` file in RStudio and render it using Quarto. This will:

* Load the processed data
* Reproduce the main statistical analyses
* Regenerate all figures produced automatically

---


## Data availability

All data underlying the analyses reported in the paper are publicly available in this repository in `.xlsx` format. The data are provided in a form suitable for inspection and reanalysis.

A concise *Data Availability Statement* suitable for journal submission is provided in `DATA_AVAILABILITY.md`.
